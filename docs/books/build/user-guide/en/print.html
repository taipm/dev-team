User Guide - @microai.club/dev-team

Welcome to the User Guide for the dev-team framework!

Introduction

@microai.club/dev-team is a Claude Code configuration framework for software development teams. This framework provides:

  • Agents - Specialized AI assistants for specific tasks
  • Teams - Groups of agents coordinating to solve complex problems
  • Commands - Slash commands to quickly invoke agents
  • Skills - Reusable capabilities for various situations

Who Is This Book For?

This book is for users who want to:

  • Install and configure dev-team in their projects
  • Use the built-in agents and teams
  • Apply effective workflows to their daily work

Note: If you want to create new agents or teams, please refer to the Developer Guide.

Beginners

  1. Start with Part 1: Getting Started
  2. Follow the 5-Minute Quickstart
  3. Explore the Built-in Agents

Experienced Users

  • Refer to Part 3: Teams to learn about multi-agent coordination
  • Check Real-World Workflows for specific use cases

settings.json

Team-shared configuration for dev-team.

Location

.claude/settings.json

Structure

{
  "permissions": {
    "allow": [
      "Bash(git:*)",
      "Bash(npm:*)",
      "WebFetch(domain:github.com)"
    ],
    "deny": [
      "Read(.env)",
      "Bash(rm -rf:*)"
    ]
  }
}

Permissions

allow

Patterns allowed without confirmation:

"allow": [
  "Bash(git:*)",        // All git commands
  "Bash(npm:*)",        // All npm commands
  "Read(**/*.go)",      // Read Go files
  "WebFetch(domain:X)"  // Fetch from domain X
]

deny

Blocked patterns:

"deny": [
  "Read(.env)",         // Don't read .env
  "Bash(rm -rf:*)"      // No recursive delete
]

Examples

Go Project

{
  "permissions": {
    "allow": [
      "Bash(go:*)",
      "Bash(git:*)",
      "Read(**/*.go)"
    ]
  }
}

Node.js Project

{
  "permissions": {
    "allow": [
      "Bash(npm:*)",
      "Bash(node:*)",
      "Bash(git:*)"
    ]
  }
}

Notes

  • This file is committed to git
  • Team members share these settings
  • Use settings.local.json for personal settings

Agent Overview

Introduction to the Agent system in the dev-team framework.

What is an Agent?

An Agent is a specialized AI assistant configured to perform specific tasks. Each agent has:

  • Persona: Role and communication style
  • Knowledge: Domain expertise
  • Tools: Allowed tools to use
  • Memory: Ability to remember across sessions

Why Use Agents?

Comparison with Plain Claude

AspectPlain ClaudeAgent
KnowledgeGeneralDeep domain expertise
ContextStarts freshHas built-in context
WorkflowGeneralSpecific process
MemoryNoneMemory system

Benefits of Agents

  1. Specialization: Agent deeply understands specific domain
  2. Consistency: Same agent always works the same way
  3. Time Saving: No need to explain context each time
  4. Extensibility: Create new agents for new needs

Agent Types

1. Specialist Agents

Domain experts:

  • npm-agent: NPM package management
  • go-dev-portable: Go development
  • go-refactor-portable: Go refactoring

2. Meta Agents

Agents that can create other agents:

  • father-agent: Creates agents, commands, knowledge

3. Thinking Agents

Agents that assist with thinking:

  • deep-question-agent: Deep questioning
