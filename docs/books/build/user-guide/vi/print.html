Hướng Dẫn Sử Dụng @microai.club/dev-team

Chào mừng bạn đến với Hướng Dẫn Sử Dụng của dev-team framework!

Giới Thiệu

@microai.club/dev-team là một framework cấu hình Claude Code cho các team phát triển phần mềm. Framework này cung cấp:

  • Agents - Các trợ lý AI chuyên biệt cho từng nhiệm vụ
  • Teams - Nhóm agents phối hợp giải quyết vấn đề phức tạp
  • Commands - Các lệnh slash để kích hoạt agents nhanh chóng
  • Skills - Các kỹ năng tái sử dụng cho nhiều tình huống

Cuốn Sách Này Dành Cho Ai?

Cuốn sách này dành cho người dùng muốn:

  • Cài đặt và cấu hình dev-team trong dự án của mình
  • Sử dụng các agents và teams có sẵn
  • Áp dụng workflows hiệu quả vào công việc hàng ngày

Lưu ý: Nếu bạn muốn tạo mới agents hoặc teams, hãy tham khảo cuốn Developer Guide.

Kiểm Tra Tổng Thể

Chạy script sau để kiểm tra tất cả requirements:

#!/bin/bash
echo "Checking requirements..."

# Node.js
if command -v node &> /dev/null; then
    NODE_VERSION=$(node --version | cut -d'v' -f2 | cut -d'.' -f1)
    if [ "$NODE_VERSION" -ge 18 ]; then
        echo "✅ Node.js: $(node --version)"
    else
        echo "⚠️ Node.js: $(node --version) (cần >= 18.0.0)"
    fi
else
    echo "❌ Node.js: Chưa cài đặt"
fi

# npm
if command -v npm &> /dev/null; then
    echo "✅ npm: $(npm --version)"
else
    echo "❌ npm: Chưa cài đặt"
fi

# Claude Code
if command -v claude &> /dev/null; then
    echo "✅ Claude Code: Đã cài đặt"
else
    echo "❌ Claude Code: Chưa cài đặt"
fi

# Git
if command -v git &> /dev/null; then
    echo "✅ Git: $(git --version)"
else
    echo "⚠️ Git: Chưa cài đặt (optional)"
fi

echo "Done!"

settings.json (Team Settings)

File này chứa cấu hình chung cho cả team, được commit vào git.

Cấu Trúc Cơ Bản

{
  "permissions": {
    "allow": [],
    "deny": []
  },
  "hooks": [],
  "env": {}
}

Permissions

Quản lý quyền truy cập tools:

{
  "permissions": {
    "allow": [
      "Bash(npm:*)",
      "Bash(git:*)",
      "Bash(pnpm:*)",
      "Read(**/*.go)",
      "Write(**/*.go)"
    ],
    "deny": [
      "Read(.env)",
      "Read(.env.*)",
      "Read(secrets/**)",
      "Read(**/*.pem)",
      "Read(**/*.key)"
    ]
  }
}

Pattern Syntax:

PatternÝ nghĩa
Bash(npm:*)Cho phép npm commands
Read(**/*.go)Cho phép đọc Go files
Write(src/**)Cho phép ghi trong src/
WebFetch(domain:github.com)Cho phép fetch từ GitHub

Hooks

Tự động hóa với hooks:

{
  "hooks": [
    {
      "type": "UserPromptSubmit",
      "command": ".microai/hooks/user-prompt-submit/log-prompt.sh"
    },
    {
      "type": "PreToolUse",
      "matcher": "Bash",
      "command": ".microai/hooks/pre-tool-use/log-bash.sh"
    }
  ]
}

Environment Variables

{
  "env": {
    "PROJECT_NAME": "my-project",
    "DEFAULT_BRANCH": "main"
  }
}

Merge Rules

Settings được merge theo thứ tự:

settings.json (base)
    ↓
settings.local.json (override)
    ↓
Final settings

Rules:

  • deny: Được cộng dồn (union)
  • env: Local override base

Agent Là Gì?

Agent là một AI assistant chuyên biệt được cấu hình để thực hiện các nhiệm vụ cụ thể. Mỗi agent có:

  • Persona: Vai trò và cách giao tiếp riêng
  • Knowledge: Kiến thức chuyên môn về lĩnh vực
  • Tools: Các công cụ được phép sử dụng
  • Memory: Khả năng ghi nhớ qua các phiên làm việc

Tại Sao Dùng Agent?

Lợi ích của Agent

  1. Chuyên môn hóa: Agent hiểu sâu về domain cụ thể
  2. Nhất quán: Cùng một agent luôn làm việc theo cách nhất quán
  3. Tiết kiệm thời gian: Không cần giải thích context mỗi lần
  4. Có thể mở rộng: Tạo agent mới cho nhu cầu mới

Các Loại Agent

1. Specialist Agents

Agent chuyên gia trong một lĩnh vực:

  • npm-agent: Quản lý npm packages
  • go-dev-portable: Phát triển Go
  • go-refactor-portable: Refactor Go code

2. Meta Agents

Agent có thể tạo ra agents khác:

  • father-agent: Tạo agents, commands, knowledge mới

3. Thinking Agents

Agent hỗ trợ tư duy:

  • deep-question-agent: Đặt câu hỏi sâu

4. **Systems Thinking**: Xem xét tương tác hệ thống
5. **Inversion**: Đảo ngược vấn đề
6. **Analogical Thinking**: Tìm pattern tương tự
7. **Pre-mortem**: Giả định thất bại và phân tích

### Cách Sử Dụng

### Ví Dụ

deep-question: Áp dụng 5 Why Analysis:

❓ Why 1: Tại sao timeout? → Response time > threshold

❓ Why 2: Tại sao response time cao? → Database queries chậm

❓ Why 3: Tại sao queries chậm? → Thiếu indexes, full table scans

❓ Why 4: Tại sao thiếu indexes? → Không có query analysis trong development

❓ Why 5: Tại sao không có query analysis? → Thiếu tooling và process

🎯 Root Cause: Thiếu observability tooling trong development process

📋 Actions:

  1. Add query logging
  2. Setup slow query monitoring
  3. Add index recommendations to CI

Phong cách: Đặt câu hỏi sâu, thách thức assumptions

  • Socratic questioning
  • Assumption mining

  • “Giả định ẩn ở đây là gì?”
  • “Nếu ngược lại thì sao?”

🧬 Aristotle - The Logician

Phong cách: Phân tích logic, categories, structures

Specialties:

  • Logical reasoning
  • Classification
  • Cause-effect analysis

Khi nào active: Phase ANALYZE

Approach:

  • Chia nhỏ vấn đề
  • Phân loại theo categories
  • Xây dựng logical framework

⚡ Musk - The Disruptor

Phong cách: First principles, phá vỡ conventions

Specialties:

  • First principles thinking
  • 10x goals
  • Convention breaking

Khi nào active: Phase EXPLORE

Câu hỏi điển hình:

  • “Nếu bắt đầu từ đầu thì làm thế nào?”
  • “Điều gì là physically impossible vs just hard?”
  • “Tại sao không gấp 10 lần?”

🔬 Feynman - The Explainer

Phong cách: Giải thích đơn giản, intuitive

Specialties:

  • Simple explanations
  • Analogies
  • First principles physics

Khi nào active: Khi cần làm rõ

Approach:

  • Giải thích như cho trẻ 12 tuổi
  • Dùng analogies quen thuộc
  • Tìm essence của vấn đề

🎭 Munger - The Sage

Phong cách: Mental models, inversion thinking

Specialties:

  • Mental models (>100)
  • Inversion
  • Multidisciplinary thinking

Khi nào active: Phase SYNTHESIZE

Models điển hình:

  • Inversion: “Làm sao để thất bại?”
  • Second-order effects
  • Circle of competence

📐 Polya - The Solver

Phong cách: Systematic problem-solving

Specialties:

  • 4-step problem solving
  • Heuristics
  • Pattern recognition

Khi nào active: Phase SOLVE

4 Steps:

  1. Understand the problem
  2. Devise a plan
  3. Carry out the plan
  4. Look back

🎨 Da Vinci - The Connector

Phong cách: Creative synthesis, cross-domain connections

Specialties:

  • Interdisciplinary thinking
  • Visual thinking
  • Creative synthesis

Khi nào active: Phase INTEGRATE

Approach:

  • Kết nối ý tưởng từ nhiều domains
  • Visual diagrams
  • Artistic perspective

## Approach 2: Binary Search + Two Pointers

### Intuition
Tìm điểm gần target nhất, rồi expand hai bên.

### Algorithm
```python
def findClosestElements(arr, k, x):
    # Binary search for closest position
    left = 0
    right = len(arr) - 1

    while right - left >= k:
        if abs(arr[left] - x) > abs(arr[right] - x):
            left += 1
        else:
            right -= 1

    return arr[left:right+1]

Correctness Proof

  • At each step, we eliminate the farther element
  • After (n-k) eliminations, k elements remain
  • These are the k closest elements

Complexity

  • Time: O(n - k) trong worst case
  • Space: O(1) excluding output

Edge Cases

  1. x smaller than all elements → first k
  2. x larger than all elements → last k
  3. x in middle → balanced expansion

Developer: 👨‍💻 Có thể optimize hơn không?

Algo: 🧮 Yes! Approach 3 uses binary search để tìm left boundary:

def findClosestElements(arr, k, x):
    left = 0
    right = len(arr) - k

    while left < right:
        mid = (left + right) // 2
        if x - arr[mid] > arr[mid + k] - x:
            left = mid + 1
        else:
            right = mid

    return arr[left:left + k]

Time: O(log(n-k) + k)

Đã thử optimize queries nhưng không đủ.

Socrates hỏi:

  • Chậm ở component nào?
  • Metrics cụ thể?
  • Đã thử gì rồi?
  • Constraints là gì?

Phase 2: ANALYZE (Aristotle, Feynman)

Aristotle phân tích:

Possible causes:
├── Database
│   ├── Query optimization
│   ├── Connection pool
│   └── Lock contention
├── Application
│   ├── Memory leaks
│   ├── Blocking operations
│   └── N+1 queries
└── Infrastructure
    ├── Network latency
    ├── Resource limits
    └── Load balancing

Feynman giải thích:

Hãy tưởng tượng như nhà hàng:
- 100 khách = 100 concurrent users
- Bếp (DB) xử lý không kịp
- Hoặc bồi bàn (app server) bận
- Hoặc cửa vào (load balancer) bottleneck

Phase 3: EXPLORE (Musk, Da Vinci)

Musk challenge:

Forget current architecture.
If rebuilding from scratch:
- Do we need a database at all?
- Can we cache 90% of requests?
- What if we went edge computing?

Da Vinci connect:

Patterns from other domains:
- CDN model for data distribution
- Circuit breaker from electrical
- Back-pressure from hydraulics

Phase 4: SYNTHESIZE (Munger, Polya)

Munger inversion:

Làm sao để HỆ THỐNG CÒN CHẬM HƠN?
- Single thread processing
- No caching
- Synchronous everywhere

→ Ngược lại: Multi-thread, cache, async

Polya systematic:

Solution Plan:
1. Add Redis cache for hot data
2. Implement connection pooling
3. Add async processing for non-critical
4. Add circuit breaker for stability

Verification:
- Load test each change
- Monitor metrics
- Rollback plan ready

Phase 5: VALIDATE (All Titans)

Final Output:

# Solution: Performance Optimization

## Root Cause
Database connection exhaustion under load
+ Synchronous processing bottleneck

## Solution
1. Redis cache (reduce DB load 70%)
2. Connection pool tuning (10→50)
3. Async queue for notifications
4. Circuit breaker for external APIs

## Implementation Order
1. [Low risk] Connection pool
2. [Medium] Redis cache
3. [Medium] Async queue
4. [Low] Circuit breaker

## Expected Improvement
- Response time: 2s → 200ms
- Capacity: 100 → 1000 concurrent users

## Risks
- Redis single point of failure
  → Mitigation: Redis cluster

## Timeline
- Week 1: Pool + Cache
- Week 2: Async + Breaker
- Week 3: Load testing

❌ Vague

✅ Specific

Reduce API response time from 2s to <200ms

Testing

Bắt đầu với happy path → edge cases

  • Save output to project docs
  • Share với team

Command Issues

Slash command không hoạt động

Triệu chứng:

Unknown command: /microai:deep-question

Giải pháp:

  1. Kiểm tra file tồn tại:

    ls .claude/commands/microai/deep-question.md

  2. Kiểm tra YAML frontmatter:

    ---
name: 'deep-question'
description: '...'
---

  3. Restart Claude Code

Agent không kích hoạt

Triệu chứng: Agent không respond như expected.

Giải pháp:

  1. Kiểm tra agent file path trong command
  2. Verify agent.md có đúng format
  3. Check permissions trong settings.json

Team Issues

Team phases không chuyển

Triệu chứng: Stuck ở một phase.

Giải pháp:

*next    # Force next phase
*status  # Check current state

Observer commands không hoạt động

Triệu chứng: *status không được recognize.

Giải pháp:

  • Đảm bảo đang trong team session
  • Dùng đúng syntax: *command (có asterisk)

Configuration Issues

Settings không được apply

Triệu chứng: Permissions không đúng.

Giải pháp:

  1. Check JSON syntax:

    cat .claude/settings.json | jq .

  2. Verify merge với settings.local.json

  3. Restart Claude Code

Tools bị denied

Triệu chứng:

Permission denied for Bash(...)

Giải pháp:

  1. Thêm vào settings.json:

    {
  "permissions": {
    "allow": ["Bash(git:*)"]
  }
}

  2. Hoặc settings.local.json cho personal

Performance Issues

Response chậm

Triệu chứng: Agent/Team respond chậm.

Giải pháp:

  • Reduce context size
  • Use simpler agent (sonnet vs opus)
  • Break task into smaller parts

Memory/Context issues

Triệu chứng: Agent “quên” context trước đó.

Giải pháp:

  • Summarize periodically
  • Use memory system
  • Start fresh session
